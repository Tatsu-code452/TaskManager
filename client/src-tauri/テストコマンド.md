# **cargo test の主な実行パターン一覧**

## 1. **通常実行（並列）**
```
cargo test
```
Rust のデフォルト。  
複数テストが **並列実行**される。

MockDatabase のようにファイルアクセスがある場合は  
**os error 32（ファイルロック）**が起きやすい。

---

## 2. **直列実行（ファイルロック対策）**
```
cargo test -- --test-threads=1
```

これが **最も重要**。  
テストが 1 つずつ順番に実行されるので、  
MockDatabase のファイルロック問題が完全に解消される。

---

## 3. **特定のテストファイルだけ実行**
```
cargo test --test defect_service_tests
```

ファイル名（拡張子なし）を指定する。

---

## 4. **特定のテスト関数だけ実行**
```
cargo test test_update
```

部分一致なので、`test_update` を含むテストが実行される。

---

## 5. **出力を全部表示（println! を見たいとき）**
```
cargo test -- --nocapture
```

テスト中の println! が全部見える。

---

## 6. **直列 + 出力表示（MockDB のデバッグに最適）**
```
cargo test -- --test-threads=1 --nocapture
```

- ファイルロックが起きない  
- println! が全部見える  
- デバッグに最適  

---

## 7. **特定のテストファイルを直列で実行**
```
cargo test --test defect_service_tests -- --test-threads=1
```

---

## 8. **特定のテスト関数を直列で実行**
```
cargo test test_update -- --test-threads=1
```
