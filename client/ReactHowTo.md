# 命名規則
- コンポーネント:PascalCase（UpperCamelCase）
- フック(UIの状態管理、ロジック):useXxx
- ドメイン、ユーティリティ:camelCase または kebab-case
- コンテキスト:XxxContext、XxxProvider

# React状態種別
- UI 入力状態=UI更新
- UI 表示状態（副作用の結果）=handler更新
- 副作用の結果状態=handler,logic更新

# React責任分離
- UI(tsx)
  - 表示 + イベント通知
  - 副作用を持たない

- handler
  - UI のための副作用を実行する
  - UI 状態を更新する
  - logic を呼び出す

-logic
  - 純粋ロジック（API / 計算）
  - UI を知らない
  - 状態を更新しない
  
- フロー
  UI (tsx)
    ↓ イベント通知
  handler
    ↓ ロジック呼び出し
  logic
    ↑ 結果を返す
  handler
    ↓ UI 状態を更新
  UI (tsx)
