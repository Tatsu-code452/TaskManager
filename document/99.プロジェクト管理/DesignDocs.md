# Design Doc  
# プロジェクト管理統合システム  

## コンテキストと背景  
現在、プロジェクト管理情報がExcelや口頭、複数のシステムに分散しており、タスク漏れや進捗遅延が頻発している。特に小規模案件では管理が行き届かず、工数集計や進捗把握に多大な時間を要している。これにより、プロジェクト全体の透明性と効率性が損なわれている。  

---

## ゴールと非ゴール  
- **ゴール**  
  - タスク・工数・進捗を一元管理できるWebアプリケーションの構築  
  - ガントチャートによる直感的な進捗可視化  
  - 工数の予実管理とレポート出力機能の提供  
  - セキュアなユーザー認証と操作ログの記録  

- **非ゴール**  
  - モバイルアプリの提供（PWA対応は将来的に検討）  
  - 外部サービス（Googleカレンダー等）との連携（初期段階では対象外）  

---

## 提案するソリューション  
- **フロントエンド**  
  - React + TypeScript によるSPA構成  
  - Viteを利用した高速開発環境  
  - UIコンポーネント：タスク一覧、ガントチャート、工数入力、管理画面  

- **バックエンド**  
  - Node.js (Express) によるAPIサーバー  
  - PostgreSQLを用いた堅牢なデータ管理  
  - トランザクション管理とエラーハンドリングを重視  

- **認証・セキュリティ**  
  - JWTベースのトークン認証  
  - XSS/CSRF対策、セッション管理  
  - 操作ログ記録による監査性の確保  

- **アーキテクチャ設計上のポイント**  
  - SPA + API構成による疎結合設計  
  - CI/CD導入による継続的デリバリー  
  - テスト設計（ユニットテスト、統合テスト）による品質保証
