# プロジェクト計画書

## プロジェクト管理統合システム

# 1. プロジェクト概要

本プロジェクトは、分散しているプロジェクト管理情報（Excel、口頭、複数システム）を統合し、タスク・工数・進捗を一元管理できるWebアプリケーションを構築することを目的とする。ガントチャートによる視覚的な進捗管理、工数の予実管理、レポート出力機能を備え、セキュアなユーザー認証と操作ログ記録を実現する。

# 2. 背景と課題

現状、プロジェクト管理のために必要な情報が分散しておりタスク漏れや進捗遅延が頻発

小規模案件では管理が行き届かず、工数集計や進捗把握に時間がかかる

プロジェクト全体の透明性と効率性が低下

# 3. ゴールと非ゴール

## ゴール

タスク・工数・進捗の一元管理

ガントチャートによる進捗可視化

工数予実管理とレポート出力

セキュアな認証と操作ログ記録

## 非ゴール

モバイルアプリ提供（将来的にPWA対応を検討）

外部サービス連携（Googleカレンダー等）

# 4. 提案するソリューション

フロントエンド: React + TypeScript によるSPA構成、Viteを利用した高速開発環境

バックエンド: Node.js (Express) + PostgreSQL、トランザクション管理とエラーハンドリングを重視

認証・セキュリティ: JWTベース認証、XSS/CSRF対策、セッション管理、操作ログ記録

UI機能: タスク一覧、ガントチャート、工数入力、管理画面

アーキテクチャ設計: SPA + API構成による疎結合設計、CI/CD導入、テスト設計（ユニット・統合テスト）

# 5. スケジュール

フェーズ1: 要件定義（2週間）

フェーズ2: 設計（3週間）

フェーズ3: 開発（8週間）

フェーズ4: テスト（4週間）

フェーズ5: リリース・運用開始（2週間）

# 6. リソース計画

人員: PM1名

環境: GitHubによるリポジトリ管理、CI/CDパイプライン

# 7. リスクと対策

リスク: 要件変更によるスケジュール遅延

対策: アジャイル開発手法を導入し、短期スプリントで進捗確認

リスク: セキュリティ脆弱性

対策: 定期的なセキュリティレビューと自動テスト導入

# 8. 成果物

Webアプリケーション（タスク管理、ガントチャート、工数管理機能搭載）

管理者向けレポート出力機能

操作ログ・監査機能

# 9. 今後の展望

PWA対応によるモバイル利用拡張

外部サービス連携（Googleカレンダー、Slack等）

AIによる工数予測や進捗分析の導入
