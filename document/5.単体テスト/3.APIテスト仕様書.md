# 案件管理システム API テスト仕様書（Tauri コマンド）

## 1. 目的
本書は、Tauri の Rust コマンド（invoke）を通じて提供される  
バックエンド API の入出力・バリデーション・データ更新が  
仕様通りに動作することを確認するためのテスト仕様書である。

## 2. テスト方針
- Tauri コマンドを直接呼び出す（HTTP は使用しない）
- JSON ファイルを DB として使用するため、テスト用にコピーして使用
- 正常系・異常系を明確に分離
- 入力 → 出力 → JSON DB の変化を確認
- 自動化可能な構造（Rust integration test / invoke mock）

---

# 3. テスト対象コマンド一覧

| コマンド名 | 機能 |
|------------|------|
| get_projects | 案件一覧取得 |
| get_project_detail | 案件詳細取得 |
| create_project | 案件作成 |
| update_project | 案件更新 |
| create_task | タスク作成 |
| update_task | タスク更新 |
| create_issue | 課題作成 |
| update_issue | 課題更新 |
| create_defect | 欠陥作成 |
| update_defect | 欠陥更新 |
| get_gantt | ガントチャートデータ取得 |
