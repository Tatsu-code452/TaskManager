# 案件管理システム バックエンド単体テスト仕様書（Rust）

## 1. 目的
本書は、案件管理システムのバックエンド（Rust）における  
**ドメインロジック・サービス層・リポジトリ層** の単体テストを網羅的に実施するための仕様書である。

対象は以下のレイヤー：

- ドメインロジック（進捗計算、日付整合性、工数計算）
- サービス層（ProjectService, TaskService, IssueService など）
- リポジトリ層（DB CRUD）
- バリデーションロジック
- DTO → Entity 変換
- Entity → JSON 変換

---

# 2. テスト方針

- **機能単位**で整理し、機能追加・変更に強い構造とする  
- 正常系・異常系を明確に分離  
- DB アクセスは mock（または in-memory SQLite）で実施  
- API レイヤーは対象外（結合テストで実施済み）  
- 各テストケースにチェック欄を設置し、レビューしやすくする  

---

# 3. テスト対象機能一覧（バックエンド）

| カテゴリ       | 機能          | 主な責務                           |
| -------------- | ------------- | ---------------------------------- |
| 案件管理       | 案件作成      | バリデーション、初期ステータス設定 |
|                | 案件更新      | ステータス遷移チェック             |
|                | 案件進捗計算  | タスク進捗の集約                   |
| マイルストーン | CRUD          | 日付整合性、ステータス更新         |
| フェーズ管理   | CRUD          | 表示順制御                         |
| タスク管理     | CRUD          | 日付整合性、工数整合性             |
|                | 進捗計算      | 計画/実績工数から進捗算出          |
| 工数管理       | 計画工数      | 日別工数合計                       |
|                | 実績工数      | 日別工数合計                       |
| 課題管理       | CRUD          | 必須項目チェック                   |
| 欠陥管理       | CRUD          | 必須項目チェック                   |
| ガントチャート | 期間→バー変換 | 日付→座標変換                      |
