# 1. テスト基本方針
構成は明確に分離されているため、テストもこの構成に分けて行う。
- UI（Page / Component） → 表示・イベント発火
- Controller → ロジック
- State Hooks → 値管理
- API → Tauri 経由の I/O

# 2. テスト対象と観点（機能網羅 & ロジック網羅）
✔ ProjectListPage（UI）
- 検索欄の入力が反映される
- 検索ボタン押下で searchProjects が呼ばれる
- クリアボタン押下で clearSearch + loadProjects が呼ばれる
- 新規作成ボタンでモーダルが開く
- 編集ボタンでモーダルが開く
- テーブルが正しく描画される

✔ useProjectListController（ロジック）
- loadProjects が API を正しく呼ぶ
- searchProjects が API を正しく呼ぶ
- clearSearch が初期値に戻す
- openCreateModal が正しい初期 form をセット
- openEditModal が選択データを form にセット
- handleSubmitCreate が create → loadProjects → closeModal を実行
- handleSubmitUpdate が update → loadProjects → closeModal を実行

✔ ProjectForm（UI）
- 入力欄が form の値を表示する
- 入力変更で onChange が呼ばれる
- 作成/更新ボタンで onSubmit が呼ばれる
- キャンセルボタンで onCancel が呼ばれる
