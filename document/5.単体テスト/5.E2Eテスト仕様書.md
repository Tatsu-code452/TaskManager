# React + Tauri E2E ワークフロー仕様書  
対象：案件管理システム（Project / Phase / Task / Issue / Defect / Gantt）

本仕様書は、React UI と Tauri（Rust バックエンド）の  
**アプリ全体の操作フローを網羅的に確認する E2E テスト** を定義する。

---

# 1. 対象範囲（Scope）

- React UI（画面遷移・入力・一覧更新）
- Tauri invoke（バックエンド呼び出し）
- Rust Command（CRUD）
- JSON DB（mock_db.json）
- ID 採番処理
- 画面間連携（Project → Phase → Task → Issue → Defect）

---

# 2. テスト観点

| 観点               | 説明                                                |
| ------------------ | --------------------------------------------------- |
| 画面遷移           | 一覧 → 詳細 → 編集 → 戻る の流れ                    |
| CRUD 操作          | create / update / delete の UI 操作                 |
| invoke 呼び出し    | 正しい payload が Rust に渡る                       |
| DB 反映            | JSON DB が正しく更新される                          |
| 一覧更新           | CRUD 後に UI が最新状態を取得する                   |
| タブ連携           | Project → Milestone → Phase → Task → Issue → Defect |
| 入力バリデーション | 必須項目のチェック                                  |
| ID 採番            | create 時に ID が自動採番される                     |

---

# 3. ワークフロー別 E2E テストケース

---

# 3.1 案件一覧（Project List）

## ✔ PL-01：案件検索  
- 操作：案件名「A」で検索  
- 期待：案件Aのみ表示

## ✔ PL-02：検索条件クリア  
- 操作：[クリア]  
- 期待：全件表示

## ✔ PL-03：新規案件作成  
- 操作：[新規案件作成] → 入力 → 保存  
- 期待：  
  - invoke("create_project") が呼ばれる  
  - 一覧に新規案件が追加される

---

# 3.2 案件詳細（Project Detail）

# 4. 補足

- Playwright を使用して UI を自動操作する  
- Tauri dev server（http://localhost:1420）を対象とする  
- JSON DB はテスト用 mock_db.json を使用  
- CRUD ロジックは Rust 側の単体テストで保証済み  
- 本仕様書では **画面操作フローの正しさ** を検証する
