# Defect（欠陥）フロントエンド／Tauri 結合テスト仕様書  
対象：React（UI）＋ Tauri（Rust バックエンド API）

本仕様書は、Defect（欠陥）に関する **フロントエンド UI と Tauri バックエンドの結合動作** を確認するためのテスト項目を定義する。

バックエンド単体テストでは CRUD のロジックを確認済みのため、  
本仕様書では **UI → Tauri → Rust の連携** を中心に検証する。

---

# 1. 対象範囲（Scope）

- React UI（フォーム入力、一覧表示、ボタン操作）
- Tauri コマンド呼び出し（invoke）
- Rust 側 DefectCommand（create / update / delete / list）
- JSON DB（mock_db.json）
- ID 採番処理（Tauri 経由での動作確認）

---

# 2. テスト観点

| 観点             | 説明                                                |
| ---------------- | --------------------------------------------------- |
| UI 入力          | 必須項目の入力有無、入力値の反映                    |
| invoke 呼び出し  | 正しいコマンド名・パラメータで呼ばれるか            |
| バックエンド応答 | 成功／失敗のレスポンスが UI に反映されるか          |
| 一覧更新         | create/update/delete 後に UI が最新状態を取得するか |
| エラーメッセージ | バリデーションエラー／NotFound が UI に表示されるか |
| ID 採番          | create 時に UI 上で採番結果が確認できるか           |

---

# 3. テストケース一覧

## 3.1 list_defect（検索）

### ✔ FE-LS-D01-01：キー情報なし → エラー表示  
- 操作：検索フォームで project_id="" のまま検索  
- 期待：  
  - invoke("list_defect", { project_id: "" }) が呼ばれる  
  - Rust 側が Err を返す  
  - UI に「キーが必要です」などのエラー表示

---

### ✔ FE-LS-D01-02：単独キー → 該当データのみ表示  
- 操作：project_id="P1" を入力して検索  
- 期待：  
  - invoke("list_defect", { project_id: "P1" })  
  - Rust 側から P1 のデータのみ返る  
  - UI の一覧が P1 のデータだけに更新される

---

## 3.2 create_defect（作成）

### ✔ FE-CR-D01-01：キー情報なし → 作成不可  
- 操作：project_id="" のまま「作成」  
- 期待：  
  - invoke は呼ばれるが Rust 側が Err  
  - UI にエラー表示  
  - 一覧は変化しない

---

### ✔ FE-CR-D01-02：キー情報のみ → 初期値で登録  
- 操作：project_id="P1" のみ入力して作成  
- 期待：  
  - invoke("create_defect", { project_id: "P1" })  
  - Rust 側で DEFECT-1 が採番される  
  - UI の一覧に DEFECT-1 が追加される  
  - 各項目は初期値（空文字／None）

---

### ✔ FE-CR-D01-03：全情報あり → 全項目を登録  
- 操作：全項目を入力して作成  
- 期待：  
  - invoke の payload が UI 入力と一致  
  - Rust 側で正しく登録  
  - UI の一覧に入力内容がそのまま反映される

---

### ✔ FE-CR-D01-04：欠番がある場合でも正しく採番  
- 前提：DB に ["DEFECT-1", "DEFECT-3"]  
- 操作：project_id="P1" で作成  
- 期待：  
  - Rust 側で DEFECT-4 が採番  
  - UI に DEFECT-4 が追加される

---

# 4. 補足

- Tauri の invoke は `window.__TAURI__.invoke("xxx", payload)` を想定  
- UI 側は React（useState / useEffect）で一覧を管理  
- バックエンド単体テストで CRUD ロジックは保証済み  
- 本仕様書では **UI → Tauri → Rust の連携** のみを検証する
